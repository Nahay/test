Models

Un model correspond à une collection qui sera créée dans la base de données. Par convention, on appelera la variable du schema nomTableSchema , mais le plus important sera le nom donné lors de l’export du schema, ici Date. Ce sera le nom donné à la collection dans la base de données.

const  DateSchema  =  Schema({
	dateC: {
		type:  Number,
		required:  true,
		unique:  true
	},
	// ...
});
module.exports  =  model('Date', DateSchema);

Clés

Comme c’est une base de données NoSQL, chaque document sera en json, donc en format clé / valeur .

Pour chaque clé, on devra spécifier son type (liste des types ici).

Ensuite, on pourra indiquer s’il est obligatoire de le rentrer : required : true . Par défaut, si on ne spécifie rien, la valeur sera false . On pourra indiquer une valeur par défaut, si rien n’est entré, par un default : valeur .

Il sera utile de spécifier si la valeur devra être unique par un unique : true , par exemple pour un nom d’utilisateur.
